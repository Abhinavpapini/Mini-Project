WHY C4 DOMINATES
=====================================================================

1. COMPLEMENTARY SSL REPRESENTATIONS:
   HuBERT-large (self-supervised):
     - Learns acoustic/phonetic structure from raw audio masking
     - Excellent at capturing spectro-temporal stutter patterns
     - Layer 21: high-level phonemic representation

   Whisper-large (ASR-supervised):
     - Trained for speech → text transcription
     - Encodes LINGUISTIC structure: word boundaries, phoneme sequences
     - Layer 28: near-output, highly text-aligned semantics
     - Stutters directly affect: word completion (Block), repetition
       boundaries (WordRep, SoundRep), filler token positions (Interjection)

2. LINGUISTIC + ACOUSTIC SYNERGY IS CRITICAL:
   WordRep F1=0.695 (prev best: B2=0.512):
     Word repetitions ("I I I want") are clearly detectable in Whisper's
     text-aligned representations — repeated token patterns at word boundaries.
     HuBERT alone misses this boundary information.

   Interjection F1=0.787 (prev: F3=0.644):
     Filler words ("um", "uh", "like") are ASR-trained OOV patterns —
     Whisper explicitly models these as distinct tokens. The cross-attention
     lets HuBERT features select the Whisper filler representations.

3. CROSS-ATTENTION ENABLES TARGETED FEATURE SELECTION:
   Rather than simply concatenating both feature vectors (which would be
   2304-dim and lose interaction structure), cross-attention lets each
   model's features query the other for relevant information:
   - HuBERT → Whisper: "which linguistic regions match these acoustic patterns?"
   - Whisper → HuBERT: "which acoustic patterns match these linguistic tokens?"

4. TRAINING DYNAMICS — IMPORTANT NOTE:
   The model exhibits SEVERE OVERFITTING after epoch 3:
   - ep3: train=0.392, val=0.412, F1=0.659 (BEST)
   - ep40: train=0.002, val=1.772, F1=0.627
   Train loss → 0.001 (memorises training set completely).
   This means the model capacity (1.28M params) is too large for the
   dataset without proper regularisation. The checkpoint at ep3 captures
   the optimal generalisation point BEFORE memorisation kicks in.

   REMEDIATION FOR FUTURE RUNS:
   - Add stronger dropout (0.5 on MLP head)
   - Add weight decay 1e-3 (current: 1e-4)
   - Add label smoothing (0.1)
   - Use early stopping with patience=5
   - Reduce learning rate warmup: lr_init=1e-4 → 3e-4 over 5 epochs
   Expected F1 with regularisation: 0.67-0.70

Key Hypotheses and Expected Outcomes
------------------------------------
- H1: Layer optimality is stutter-type dependent: Blocks (motor planning failure)
  may be better captured in later layers encoding prosodic structure, while sound
  repetitions (acoustic-phonetic events) may peak in mid-layers. Expected: per-type
  optimal layers differ by 3 to 8 layers.
- H2: Dimensionality reduction improves generalization: PCA to about 32 dimensions
  from 1024 should improve macro-F1 on minority stutter types by removing noise
  dimensions, similar to the 32-dim result in the dysarthria paper.
- H3: Handcrafted stutter features provide complementary signal: Features like
  jitter, pause ratio, and speaking rate should show low correlation with compact
  SSL+MFCC fusion (similar to <0.1 mean correlation in the dysarthria paper),
  indicating additive value.
- H4: Graph-based models outperform CNN on temporal stutter patterns: Disfluencies
  span multiple frames and involve non-local acoustic dependencies (for example,
  coarticulation changes before a block). GCN/D-GAT are expected to capture these
  better than frame-averaged CNN classifiers.
- H5: Multi-label formulation improves co-occurrence handling: Converting to
  multi-label binary classification should improve F1 on clips with two or more
  simultaneous disfluency annotations, expected to be a significant portion of
  SEP-28k.
