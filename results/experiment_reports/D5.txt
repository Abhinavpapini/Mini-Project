=====================================================================
MODEL CONFIGURATION
=====================================================================

Feature input:
  - HuBERT-large layer 21 (1024-dim → PCA-32, 58.3% explained variance)
  - MFCC-21 (first 21 cols from 78-dim stats cache, standardised)
  - Fused dim: 53 (same as F1 / F4)

Architecture: D5 Multi-task CNN
  Shared backbone (CNN-1D):
    Conv1d(1→64, k=7) + BN + GELU
    Conv1d(64→128, k=5) + BN + GELU
    Conv1d(128→256, k=3) + BN + GELU → AdaptiveAvgPool(1)
    Flatten → Dropout(0.2) → Linear(256→128) → GELU   [shared repr]
  Head 1 (detect): Dropout → Linear(128→64) → GELU → Linear(64→1)
  Head 2 (type)  : Dropout → Linear(128→64) → GELU → Linear(64→5)
  Total params: 190,854

Loss:
  L_detect : BCEWithLogitsLoss (pos_weight=16816/4090≈4.11 — upweight stutter)
  L_type   : MultiLabelSoftMarginLoss (stutter samples only, fluent masked out)
  L_total  : 1.0 × L_detect + 1.0 × L_type
  Threshold: 0.5

Sampler: WeightedRandomSampler by fluent/stutter ratio (oversample fluent)

Training: AdamW (lr=3e-4, wd=1e-4), CosineAnnealingLR, 30 epochs, batch=256

Training data split:
  16816 stutter clips — both heads learn from these
  4090 fluent clips  — only detection head learns (type loss masked)

=====================================================================
TRAINING DYNAMICS
=====================================================================

Epochs 1-8: Detection head collapses (det_f1≈0) — all predictions negative.
  Detection pos_weight=4.11 is insufficient to overcome early class imbalance.
  Type head also low (macro_f1=0.11-0.22) due to gradient interference.

Epochs 9-19: Both heads improve together. Val_loss drops from 1.25 → 0.92.
  Detection F1 slowly rises: 0.24 @ ep9 → 0.53 @ ep19 (best epoch).
  Type macro_f1 climbs to 0.489 @ ep19.

Epochs 20-30: Plateau/fluctuations without further gain.
  This suggests epoch 19 is the sweet spot for the combined loss.

Key observation: Detection head learns MUCH slower than a standalone binary
classifier would, because the shared backbone is being pulled toward type
classification objectives simultaneously (gradient conflict).

=====================================================================
KEY INTERPRETATIONS
=====================================================================

1. MULTI-TASK LEARNING HURTS TYPE CLASSIFICATION (-0.06 vs F1):
   Sharing encoder parameters between detection and type head enforces a
   compromise representation. The detection task (simpler, binary) dominates
   early training, slowing type head learning. F1's single-task CNN can
   specialise the backbone entirely for type classification.

2. DETECTION HEAD IS CLINICALLY VALUABLE (AUROC=0.811):
   Despite low F1 at threshold=0.5, AUROC=0.811 shows the model orders
   clips by stutter probability well. At lower threshold (e.g., 0.3), recall
   would improve significantly — useful for clinical screening where false
   negatives are more costly than false positives.

3. DETECTION PRECISION=0.970 IS EXTRAORDINARY:
   When D5 predicts a clip as stuttering at threshold=0.5, it is correct
   97% of the time. This "high confidence positive" mode may be useful in
   clinical annotation pipelines to pre-filter reliable stutter instances.

4. DETECTION RECALL=0.363 IS THE WEAK POINT:
   At 0.5 threshold, 63.7% of stutter clips are missed. This is the key
   failure of D5's detection head — too conservative. Lowering threshold
   or upweighting L_detect (e.g., λ_detect=2.0) could help.

5. TYPE HEAD SHOWS GRADIENT INTERFERENCE CLEARLY:
   Block and Interjection (most represented classes) achieve F1=0.577/0.580,
   but WordRep and Prolongation (rare, need focused learning) suffer most
   (0.393/0.440). In single-task F1, these were 0.415/0.583. Single-task
   better at both.

6. TASK WEIGHTING NOT OPTIMAL (λ=1.0 each):
   With 16816 stutter and 4090 fluent, detection loss is naturally smaller
   (easy majority-class samples). Type loss is larger. The equal λ may
   under-weight detection. Try λ_detect=2.0 for better detection recall.

=====================================================================
CONCLUSION
=====================================================================

Multi-task D5 did NOT outperform single-task F1 on type classification
(0.489 vs 0.550). The detection head provides unique value (AUROC=0.811)
not available from any other experiment, but at the cost of type accuracy.

Recommended use case for D5:
  - Detection stage in a 2-stage pipeline: D5 detect → C5/F1 classify
  - Clinical screening with high-precision stutter flagging (pre=0.97)
  NOT recommended as the primary type classifier (use F1 or D1 instead).

Key Hypotheses and Expected Outcomes
------------------------------------
- H1: Layer optimality is stutter-type dependent: Blocks (motor planning failure)
  may be better captured in later layers encoding prosodic structure, while sound
  repetitions (acoustic-phonetic events) may peak in mid-layers. Expected: per-type
  optimal layers differ by 3 to 8 layers.
- H2: Dimensionality reduction improves generalization: PCA to about 32 dimensions
  from 1024 should improve macro-F1 on minority stutter types by removing noise
  dimensions, similar to the 32-dim result in the dysarthria paper.
- H3: Handcrafted stutter features provide complementary signal: Features like
  jitter, pause ratio, and speaking rate should show low correlation with compact
  SSL+MFCC fusion (similar to <0.1 mean correlation in the dysarthria paper),
  indicating additive value.
- H4: Graph-based models outperform CNN on temporal stutter patterns: Disfluencies
  span multiple frames and involve non-local acoustic dependencies (for example,
  coarticulation changes before a block). GCN/D-GAT are expected to capture these
  better than frame-averaged CNN classifiers.
- H5: Multi-label formulation improves co-occurrence handling: Converting to
  multi-label binary classification should improve F1 on clips with two or more
  simultaneous disfluency annotations, expected to be a significant portion of
  SEP-28k.
