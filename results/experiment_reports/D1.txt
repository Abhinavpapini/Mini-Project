=====================================================================
MODEL CONFIGURATION
=====================================================================

Feature input:
  - HuBERT-large layer 21 (1024-dim → StandardScaler → PCA-32)
  - Explained variance retained: 58.3%
  - Input dim: 32 (pure SSL, no MFCC, no handcrafted features)

Architecture: D1 4-block Conformer multi-label
  - Each of 32 PCA components treated as a "token" position
  - Input: [B, 32] → unsqueeze → [B, 32, 1] → Linear(1→128) → [B, 32, 128]
  - 4× ConformerBlock:
      FF(half-step, expansion=4) → MHSA(8 heads, d=128) → DepthwiseConv(k=7) → FF(half-step) → LN
  - AdaptiveAvgPool over 32 positions → [B, 128]
  - LN → Dropout(0.1) → Linear(128→256) → GELU → Dropout(0.1) → Linear(256→5)
  - Total params: 1,561,093  (8.9× more than F4's 202K)

Loss: MultiLabelSoftMarginLoss (PyTorch built-in)
  = -1/C * Σ_c [y_c * log(σ(x_c)) + (1-y_c) * log(1-σ(x_c))]
  No pos_weight, no focal term. Uniform treatment of all classes.

Sampling: WeightedRandomSampler (same as F1/F4 for fair comparison)

Training:
  - Optimizer  : AdamW (lr=3e-4, wd=1e-4)
  - Scheduler  : CosineAnnealingLR (30 epochs, eta_min=1.5e-5)
  - Batch size : 256
  - Threshold  : 0.5
  - Train/test : 80/20 stratified on Block, seed=42

=====================================================================
TRAINING DYNAMICS
=====================================================================

Notable: epoch 1 macro_f1=0.171 (collapsed to random/majority) then recovered to
0.459 by epoch 2. This is a known issue with Conformer initialisation — attention
heads need one full forward pass to establish useful patterns. The model stabilises
from epoch 3 onwards.

Val loss plateau: starts at 0.561 → reaches ~0.493 by epoch 7 → stays flat
0.493-0.508 for remainder. Macro-F1 oscillates between 0.506-0.548 throughout.
High variance in macro-F1 despite stable loss indicates the decision boundary is
near the 0.5 sigmoid threshold for many clips.

Best epoch = 18 (not close to epoch 30) — suggests that with this input bottleneck
(PCA-32), 30 epochs is sufficient and more training doesn't help.

=====================================================================
KEY INTERPRETATIONS
=====================================================================

1. CAPACITY ≠ PERFORMANCE: D1 has 1.56M parameters (vs F4's 202K) but achieves
   nearly identical macro-F1 (0.548 vs 0.545). The bottleneck is the 32-dim PCA
   input — no amount of model depth can recover information lost in PCA compression.

2. LOSS FUNCTION SHAPES PRECISION/RECALL TRADE-OFF: MultiLabelSoftMarginLoss
   gives balanced precision (0.574) vs recall (0.526). Focal loss in F1 gave
   recall=0.726 / precision=0.458. For clinical screening (recall preferred),
   F1 is better. For annotation efficiency (precision preferred), D1 is better.

3. BEST AUPRC (0.602 vs 0.581 F1, 0.561 F4): D1's probability scores are better
   calibrated than F1/F4 across all threshold values. The uniform MLSM loss
   without class weighting trains the model to output more reliable confidence.

4. WORD REP GAIN (+0.028 vs F1): D1 improves WordRep F1 from 0.415 (F1) to 0.443.
   WordRep precision rises from 0.285 (F1) to 0.476 (D1) — 67% precision gain.
   This suggests that the 4-block Conformer's deeper self-attention over PCA
   features captures subtle patterns that distinguish WordRep from fluent speech,
   even though overall recall drops.

5. INTERJECTION STILL EASIEST (F1=0.625), WORDREP STILL HARDEST (F1=0.443):
   The difficulty ordering is identical across all multi-label experiments,
   confirming it is driven by class inherent difficulty (acoustic ambiguity +
   class imbalance), not loss function or architecture.

6. AUGMENTING INPUT: The next logical step is to add MFCC back (as in F1/F4) or
   use a different SSL layer. D1's pure PCA-32 is a clean ablation confirming that
   handcrafted features (MFCC-21 in F1/F4) provide the marginal gains, not just
   Conformer depth.
