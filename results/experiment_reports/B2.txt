=====================================================================
ROOT CAUSE — WHY KPCA FAILS ON SSL FEATURES
=====================================================================

1. SSL FEATURES ALREADY NON-LINEAR — KPCA REDUNDANT:
   HuBERT features are the output of 24 Transformer layers with
   non-linear activations. The feature manifold is already highly
   non-linear and task-discriminative. Applying another non-linear
   projection (KPCA-RBF) distorts this carefully learned structure
   rather than enhancing it.

2. TRAINING INSTABILITY (F1 OSCILLATING 0.03→0.39 ACROSS 30 EPOCHS):
   The KPCA-32 embedding is much harder for the CNN to learn from than
   linear PCA. Kernel eigenvectors don't have explicit variance ordering
   (unlike PCA's guaranteed max-variance first component), making the
   loss landscape less smooth.

3. PROLONGATION CATASTROPHIC FAILURE (F1=0.173):
   Prolongation's acoustic signature (stretched phonemes) is encoded in
   fine-grained spectral structure. The KPCA compression at γ=0.00121
   with only 32 components loses this structure — K(x,y)≈0.14 means
   samples must be very similar to contribute to the same component.
   Prolongation clips span a wider acoustic range than other stutter types.

4. SUBSAMPLE FITTING MISMATCH (5000 of 16724 train):
   KPCA fitted on only 5000 samples may miss the global structure.
   When transforming all 20906 samples, out-of-sample extrapolation
   degrades for samples far from the 5000 fit points.

5. KPCA (γ=0.001212): Expected K≈0.135 — kernel too localised:
   With K=0.135, each point only strongly "belongs to" its very close
   neighbours in the kernel space. The 32 eigenvectors capture 32 very
   localised neighbourhoods, not global stutter-type structure.

=====================================================================
KEY FINDING
=====================================================================

⭐ IMPORTANT: Non-linear kernel compression (KPCA-RBF) DOES NOT improve
   over linear PCA when applied to deep SSL features (HuBERT-large).

   Reason: Deep neural SSL features have already learned non-linear
   acoustic representations. Linear PCA is sufficient for dimensionality
   reduction at this stage — the non-linearity was done by HuBERT.

   This result mirrors E3's finding: additional complexity in feature
   processing (hyperedges, kernel PCA) hurts when the upstream model
   (HuBERT / k-NN) already captures the right structure.

Key Hypotheses and Expected Outcomes
------------------------------------
- H1: Layer optimality is stutter-type dependent: Blocks (motor planning failure)
   may be better captured in later layers encoding prosodic structure, while sound
   repetitions (acoustic-phonetic events) may peak in mid-layers. Expected: per-type
   optimal layers differ by 3 to 8 layers.
- H2: Dimensionality reduction improves generalization: PCA to about 32 dimensions
   from 1024 should improve macro-F1 on minority stutter types by removing noise
   dimensions, similar to the 32-dim result in the dysarthria paper.
- H3: Handcrafted stutter features provide complementary signal: Features like
   jitter, pause ratio, and speaking rate should show low correlation with compact
   SSL+MFCC fusion (similar to <0.1 mean correlation in the dysarthria paper),
   indicating additive value.
- H4: Graph-based models outperform CNN on temporal stutter patterns: Disfluencies
   span multiple frames and involve non-local acoustic dependencies (for example,
   coarticulation changes before a block). GCN/D-GAT are expected to capture these
   better than frame-averaged CNN classifiers.
- H5: Multi-label formulation improves co-occurrence handling: Converting to
   multi-label binary classification should improve F1 on clips with two or more
   simultaneous disfluency annotations, expected to be a significant portion of
   SEP-28k.
