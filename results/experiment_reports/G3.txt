G3 Experiment Report

Experiment: G3 — Multi-SSL layer probes + per-class Gumbel routing
Date: 2026-05-18
Scripts: `experiments/G3_part1_probe.py`, `experiments/G3_part2_model.py`

What we tried
- Transform SEP-28k multi-label labels into a single 6-class task
- Probe each cached SSL layer with logistic regression, pick top-5 layers
- For each model: learn a softmax-weighted sum of those top layers
- Combine model streams with a small 1D-CNN and use a per-class Gumbel
  Softmax to route which model is used for each class

Data and setup
- Dataset: SEP-28k (labels only)
- Samples: 20,906 (train 16,724 / test 4,182, stratified)
- Classes: Block, Prolongation, SoundRep, WordRep, Interjection, Fluent
- Models used: `hubert-large`, `whisper-large` (other two planned models had no cached features)
- Top-K layers: 5 per model, projection dim: 256, epochs: 40, batch size: 256

Stage 1 — Layer probes (logistic regression)
- HuBERT best layer: L21 (top-5: 6,12,18,21,23). Best probe macro-F1 ≈ 0.373
- Whisper best layer: L31 (top-5: 8,16,24,28,31). Best probe macro-F1 ≈ 0.413

Stage 2 — G3 training
- Architecture: per-model WeightedLayerSum → per-model linear projection → CNN-1D →
  per-class Gumbel router → per-class heads. Total params ≈ 1.12M
- Training: 40 epochs, τ annealed 2.0 → 0.5
- Best validation macro-F1: 0.3849 (epoch 40)

Final test results (best checkpoint)
- Accuracy: 0.468
- Macro-F1: 0.382

Per-class F1 (test)
- Block: 0.479
- Prolongation: 0.306
- SoundRep: 0.156
- WordRep: 0.273
- Interjection: 0.544
- Fluent: 0.534

What the model learned
- Gumbel routing picks HuBERT for acoustic/phonetic classes (Block, Prolongation, SoundRep)
  and Whisper for word/boundary/linguistic classes (WordRep, Interjection, Fluent).
- Learned layer weights are fairly uniform across the top-5 layers; L21 (HuBERT)
  and L31 (Whisper) are slightly strongest.

Why the macro-F1 is relatively low
1) Task is harder than prior experiments — we moved from a 5-class multi-label
   setup to a 6-class single-label softmax (added the Fluent class). Macro-F1
   averages all classes equally, so minority classes hurt the average.
2) Severe class imbalance — SoundRep and WordRep have very few samples (~700 each).
3) Only two SSL models were available for this run (wav2vec2-large and wavlm-large
   were skipped because their feature caches were missing). That reduces representational diversity.
4) Training was still slowly improving at epoch 40; running longer (60–80 epochs)
   would likely improve scores.

Recommendations / next steps
- Cache features for `wav2vec2-large` and `wavlm-large` and re-run the full 4-model setup.
- Try stronger imbalance handling (focal loss, class-specific augmentation) for SoundRep/WordRep.
- Sweep more layers (not only top-5) so the WeightedLayerSum has more diverse choices.
- Train longer (60–80 epochs) to reach fuller convergence.
