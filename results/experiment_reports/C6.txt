=====================================================================
MODEL CONFIGURATION
=====================================================================

Stream 1 — HuBERT-large:
  - Model    : facebook/hubert-large-ll60k (layer 21 / 24)
  - Input dim: 1024 → PCA-32 (train-only, expl. variance ~58.2%)

Stream 2 — WavLM-SV (speaker embeddings):
  - Model    : microsoft/wavlm-base-plus-sv (94M params)
  - Method   : Mean-pool last hidden state [B, T_frames, 768] → [B, 768]
  - Input dim: 768 → PCA-32 (train-only, expl. variance ~66.9%)
  - Captures : Speaker identity, vocal tract characteristics (VoxCeleb trained)

Fusion strategy:
  - Concatenation: [HuBERT-PCA32 | WavLM-SV-PCA32] = 64-dim
  - Architecture : CNN-1D binary classifier
      Conv1d(1→32, k=7) + BN + GELU
      Conv1d(32→64, k=5) + BN + GELU
      Conv1d(64→128, k=3) + BN + GELU → AdaptiveAvgPool(1)
      Flatten → Dropout(0.3) → Linear(128→64) → GELU → Linear(64→2)
  - Total params : 44,098

Training:
  - Loss     : CrossEntropyLoss (neg/pos class weight)
  - Optimizer: AdamW (lr=3e-4, wd=1e-4)
  - Scheduler: CosineAnnealingLR (20 epochs, eta_min=1.5e-5)
  - Batch    : 256, seed=42, 80/20 stratified split

=====================================================================
KEY INTERPRETATIONS
=====================================================================

1. SPEAKER IDENTITY ADDS LIMITED VALUE: Adding WavLM-SV speaker embeddings to
   HuBERT-large PCA-32 gives mean F1=0.5617 — below C5's 0.6282 (Whisper) and
   also below F1's 0.5497 (focal loss, multi-label, same HuBERT representation).
   Speaker identity does not provide a strong signal for stutter type classification.

2. BLOCK IS MOST SPEAKER-SENSITIVE: C6's Block F1 (0.622) is closest to C5 (0.631),
   suggesting that speaker-specific vocal characteristics (e.g. laryngeal tension,
   airflow patterns) slightly help Block detection. Motor-based stutter types may be
   more speaker-specific than linguistic ones.

3. INTERJECTION AND WORD REP LEAST SPEAKER-SENSITIVE: WordRep has the largest gap
   vs C5 (-0.10). Interjection, despite being a linguistic event, also loses more
   vs C5 than Block (-0.09). This suggests Whisper's linguistic ASR features are
   far more informative than speaker identity for these classes.

4. HIGH RECALL / LOW PRECISION PATTERN: Block recall=0.838 but precision=0.495 —
   the model is over-triggering false positives. Block is acoustically heterogeneous
   (many speakers), and speaker embeddings alone can't distinguish blocked speech
   from normal variation between speakers.

5. VAL F1 VOLATILE (NOISY TRAINING): Several targets show large epoch-to-epoch
   fluctuations (e.g. Block ep6=0.51, ep7=0.62, ep8=0.58). This is typical for
   small CNN models on 64-dim input — the optimisation landscape is noisy.

6. F3 DEPENDENCY NOTED: The tracker lists C6 as a prerequisite for F3 (adversarial
   speaker disentanglement). The C6 speaker embedding cache (WavLM-SV 768-dim) can
   be reused directly for F3's GRL-based speaker de-biasing experiments.

Key Hypotheses and Expected Outcomes
------------------------------------
- H1: Layer optimality is stutter-type dependent: Blocks (motor planning failure)
   may be better captured in later layers encoding prosodic structure, while sound
   repetitions (acoustic-phonetic events) may peak in mid-layers. Expected: per-type
   optimal layers differ by 3 to 8 layers.
- H2: Dimensionality reduction improves generalization: PCA to about 32 dimensions
   from 1024 should improve macro-F1 on minority stutter types by removing noise
   dimensions, similar to the 32-dim result in the dysarthria paper.
- H3: Handcrafted stutter features provide complementary signal: Features like
   jitter, pause ratio, and speaking rate should show low correlation with compact
   SSL+MFCC fusion (similar to <0.1 mean correlation in the dysarthria paper),
   indicating additive value.
- H4: Graph-based models outperform CNN on temporal stutter patterns: Disfluencies
   span multiple frames and involve non-local acoustic dependencies (for example,
   coarticulation changes before a block). GCN/D-GAT are expected to capture these
   better than frame-averaged CNN classifiers.
- H5: Multi-label formulation improves co-occurrence handling: Converting to
   multi-label binary classification should improve F1 on clips with two or more
   simultaneous disfluency annotations, expected to be a significant portion of
   SEP-28k.
